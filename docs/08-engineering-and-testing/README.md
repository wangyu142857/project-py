# 08 Engineering And Testing

## 模块目标

从“会写功能”升级到“能维护项目”，补齐项目结构、配置管理、日志和测试等工程能力。

## 学习重点

- 分层设计
- 配置管理
- 日志规范
- 错误处理
- 统一响应结构
- 单元测试
- 接口测试
- `pytest` 与 mock 基础

## 实践任务

- 设计一个清晰的项目目录结构
- 为关键业务补测试
- 增加统一异常处理和日志
- 梳理环境配置方式

## 完成标准

- 项目结构清晰
- 关键路径具备测试覆盖
- 出现问题时有日志可查

## 后续可补充

- 目录规范
- 测试样例
- 日志规范
- 错误码设计
