# 11 Project Practice

## 模块目标

通过完整项目实践，把前面的技术点串成真实交付能力。

## 建议项目

- 项目 1：`Todo API`
- 项目 2：`博客 / CMS`
- 项目 3：`任务协作系统`

## 学习重点

- 需求拆解
- 数据建模
- API 设计
- 前后端联调
- 权限控制
- 日志与错误处理
- 缓存与异步任务
- 部署与文档

## 实践任务

- 为每个项目定义范围与核心功能
- 按阶段实现 MVP
- 逐步补齐工程化能力
- 完成至少一个可部署项目

## 完成标准

- 至少完成 2 到 3 个完整项目
- 至少 1 个项目具备上线能力
- 能清晰讲述项目设计与技术取舍

## 后续可补充

- 项目列表
- 需求文档
- 里程碑计划
- 复盘记录
