
# Python 学习大纲

## 一、阶段目标

### 目标 1：补齐后端基础能力

从“会调接口”升级到“会设计和实现接口”。

### 目标 2：具备完整项目开发能力

能够独立完成前端、后端、数据库、鉴权、部署的全链路开发。

### 目标 3：具备初级到中级全栈工程能力

不仅能写功能，还能关注结构、性能、安全、测试和上线交付。

---

## 二、学习模块总览

整个学习分为 8 个模块：

1. Python 语言基础
2. Web 与后端基础
3. Python Web 框架开发
4. 数据库与 ORM
5. 认证、权限与安全
6. 工程化与测试
7. 缓存、异步与性能优化
8. 部署与上线实战

---

## 三、详细学习大纲

### 模块 1：Python 语言基础

#### 学习目标

建立 Python 编码能力，能独立写业务逻辑代码。

#### 学习内容

##### 1. 基础语法

* 变量、数据类型
* 字符串、列表、元组、字典、集合
* 条件判断
* 循环
* 函数定义与调用

##### 2. 进阶语法

* 参数类型：位置参数、默认参数、可变参数、关键字参数
* 匿名函数
* 列表推导式 / 字典推导式
* 装饰器
* 生成器与迭代器

##### 3. 面向对象

* 类和对象
* 继承
* 多态
* 封装
* 类方法、静态方法、实例方法

##### 4. 工程基础

* 模块与包
* 虚拟环境
* pip 依赖管理
* requirements.txt
* typing 类型注解
* 异常处理
* 文件读写
* 日志基础

#### 输出要求

* 能写中小型 Python 脚本
* 能把业务逻辑拆成函数和类
* 能看懂常见 Python 后端项目的基础代码结构

---

### 模块 2：Web 与后端基础

#### 学习目标

理解服务端开发的核心概念，建立完整请求链路认知。

#### 学习内容

##### 1. HTTP 基础

* 请求方法：GET、POST、PUT、PATCH、DELETE
* 请求头、响应头
* 状态码
* 请求体与响应体
* JSON、FormData、文件上传

##### 2. Web 应用基础

* 浏览器与服务端通信流程
* 前后端分离架构
* RESTful API 设计规范
* API 参数设计
* 分页、筛选、排序
* 错误码设计

##### 3. 身份与状态管理

* Cookie
* Session
* Token
* JWT
* CORS 原理

#### 输出要求

* 能独立设计一套基本 API
* 能说清楚登录态是如何维持的
* 能理解前端请求在后端如何被处理

---

## 七、你的重点学习原则

对于前端转全栈，这几个原则很重要：

### 1. 先主线，后扩展

先把 **FastAPI + PostgreSQL + SQLAlchemy + Redis + Docker** 这一条主线打通，不要一开始学太多框架。

### 2. SQL 必须重视

数据库能力会直接决定你后端成长上限。

### 3. 项目驱动学习

不要碎片化学。每学一个模块，都尽量立刻用于项目中。

### 4. 工程能力和功能开发同样重要

后端不是只会写接口，还要会组织代码、排查问题、部署上线。
